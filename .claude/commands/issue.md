# issue

.docs/PBL 폴더에서 가장 최신 주차 문서를 읽고, 해당 내용을 기반으로 GitHub 이슈를 생성하고 브랜치를 만든다.

## 실행 순서

1. `.docs/PBL/` 폴더에서 가장 높은 번호의 week 파일을 찾는다 (예: week2.md)
2. 해당 파일을 읽고 미션 목표, 기능 요구사항, 결과물을 파악한다
3. 이슈 제목: `[WeekN] {미션 핵심 요약}` 형식으로 작성
4. 이슈 본문: 아래 형식으로 작성
   - ## 목표 (과제 목표 요약)
   - ## 구현 항목 (체크리스트 형태로 기능 요구사항 나열)
   - ## 참고 (제약사항, 보너스 과제 등)
5. `gh issue create`로 이슈를 생성한다
6. 생성된 이슈 번호로 브랜치를 만든다: `feat/{이슈번호}-week{N}-implementation`
7. 완료 후 이슈 URL과 브랜치명을 알려준다

## 규칙

- 이슈 내용은 PBL 문서를 요약하되 체크리스트 형식으로 구체적으로 작성
- 브랜치명은 영어 소문자 + 하이픈
- Co-Authored-By 태그 없이 처리한다
