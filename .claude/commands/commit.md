# commit

현재 변경 내용을 분석해서 적절한 커밋 메시지를 작성하고 커밋한다. 푸시는 하지 않는다.

## 실행 순서

1. `git status`로 변경된 파일 목록을 파악한다
2. `git diff`로 실제 변경 내용을 확인한다
3. 변경 내용을 분석해서 커밋 타입과 메시지를 결정한다
4. 변경된 파일을 모두 스테이징한다 (`git add`)
5. 커밋 메시지 형식: `{type}: {한글 설명} (#{이슈번호})`
   - 이슈 번호는 현재 브랜치명에서 추출 (예: feat/4-week2 → #4)
   - 브랜치에 이슈 번호가 없으면 이슈 번호 생략
6. `git commit`으로 커밋한다

## 커밋 타입

- `feat`: 새 기능 추가
- `fix`: 버그 수정
- `refactor`: 리팩토링 (기능 변경 없음)
- `test`: 테스트 추가/수정
- `chore`: 설정, 의존성 등 기타 변경

## 규칙

- 커밋 메시지는 한글로 작성
- 여러 관심사가 섞인 경우 가장 핵심적인 변경을 기준으로 메시지 작성
- Co-Authored-By 태그 없이 커밋한다
- 푸시는 절대 하지 않는다
