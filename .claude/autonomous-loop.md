# 자율 루프 지침

## 목표
KIT 바이브코딩 공모전 심사 기준 40점 만점 달성. 매 라운드마다 평가 → 최저 점수 항목 수정 → 재평가.

## 라운드별 진행

### 매 라운드 시작 시
1. `docs/judging-criteria.md` 읽기
2. 현재 상태 평가 (각 항목 점수 부여)
3. 가장 낮은 점수 영역 파악
4. 해당 영역 수정 작업 실행
5. `npm run build` 통과 확인
6. `git add + commit + push`
7. 점수 업데이트

### 작업 우선순위
1. **README.md** — 없으면 최우선 생성 (심사위원 첫인상)
2. **기능 동작** — 데모 플로우 깨진 곳 수정
3. **UI/UX 일관성** — 스크린샷으로 확인하며 수정
4. **문서화** — AI 활용 로그, .env.example
5. **디자인 폴리시** — 세부 간격/색상 통일

### 컨텍스트 관리
- 컨텍스트 60% 도달 예상 시:
  1. MEMORY.md + 메모리 파일 업데이트
  2. CLAUDE.md 업데이트
  3. docs/judging-criteria.md에 현재 점수 기록
  4. 자동 컴팩트 후 이어서 진행

### 토큰 소진 시
- ScheduleWakeup 15분(900초) 간격으로 재시도
- reason에 "토큰 리필 대기" 명시

### Git 규칙
- 작업 단위별 커밋 (한 번에 몰아서 X)
- 커밋 메시지 한글
- main 브랜치 직접 푸시 (마감 임박)

### 금지 사항
- 동작하는 기능 깨뜨리지 않기
- 불필요한 리팩토링 하지 않기
- 사용자가 명시적으로 승인하지 않은 대규모 구조 변경 금지
